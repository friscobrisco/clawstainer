Run untrusted AI agents safely in disposable sandboxes

No Docker. Boots in seconds. Local-first isolated Linux environments — containers or microVMs — spun up and torn down from a single CLI.

terminal
# Create a sandbox in seconds $ clawstainer create --name agent-box --memory 2048 # ✓ Created agent-box (nspawn) in 2.3s # Provision with an AI agent template $ clawstainer provision <id> --components claude-code # ✓ Provisioned claude-code in agent-box # Done? Tear it down $ clawstainer destroy <id> # ✓ Destroyed agent-box
01 Why Clawstainer

Not Docker. Not slow. Not remote.

Two runtime backends. Zero cloud dependency. Every sandbox is a fresh Linux environment with full package management, file I/O, and networking — isolated by default.

⚡

125ms boot time

Firecracker microVMs boot faster than you can blink. Nspawn containers take 2-3 seconds. Both faster than pulling a Docker image.

{ }

Hardware isolation

Firecracker gives each sandbox its own kernel. Nspawn uses namespace/cgroup isolation. Pick the security level your agents need.

→

Runs on your machine

Native on Linux. Works on macOS via Lima — one brew install away. No cloud account, no billing dashboard, no cold starts.

◇

Snapshot & reuse

Provision a sandbox once, snapshot it, spin up clones instantly. No re-downloading packages every time.

⊞

Fleet management

Deploy 10, 50, or 200 agent sandboxes from a single YAML definition. Parallel provisioning. Destroy by group or all at once.

☰

Agent templates

Built-in provisioning for Claude Code, Hermes, and OpenClaw. One command to go from empty box to running agent.

03 Use Cases

What you build with disposable sandboxes

Benchmark agent performance across LLMs

Spin up identical sandboxes for GPT-5, Claude, Gemini, and Llama — same environment, same task, different models. Compare token efficiency, tool use accuracy, and task completion rates in true isolation. No cross-contamination between runs.

evaluation
benchmark.yaml
machines: - name: bench-gpt5 count: 3 memory: 2048 provision: openclaw - name: bench-claude count: 3 memory: 2048 provision: claude-code

Give everyone on your team their own agent

Each engineer, PM, or designer gets a persistent sandbox with their preferred AI agent pre-provisioned. Personal environments that don't step on each other — with resource limits enforced per box.

team infra

Run untrusted code from LLM outputs

Let agents write and execute code without risking your host machine. Every sandbox is disposable — if an agent rm -rfs everything, destroy it and spin up a fresh one.

safety

Ephemeral CI environments

Replace heavyweight Docker-in-Docker CI with lightweight Clawstainer sandboxes. Snapshot your base image once, clone it for each job. Faster, simpler, more isolated.

devops

Local AI agent development

Build and test agents without cloud costs. Port-forward into the sandbox, iterate fast, snapshot when you have something working. Move to production when ready.

04 Fleet Management

One YAML. Hundreds of agents.

Define your fleet in a declarative config. Clawstainer creates all machines first, then provisions them in parallel batches. Destroy by group or nuke everything.

fleet.yaml
machines: - name: hermes-worker count: 3 memory: 2048 cpus: 2 provision: hermes-agent linger: true - name: openclaw count: 10 memory: 1024 cpus: 2 provision: openclaw linger: true
terminal
$ clawstainer fleet create --file fleet.yaml --parallel 5 # ✓ Created 13 machines # ✓ Provisioning [5/13]... # ✓ All 13 machines provisioned $ clawstainer fleet destroy --all # ✓ Destroyed 13 machines
